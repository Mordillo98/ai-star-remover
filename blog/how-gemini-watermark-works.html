How the watermark works

Why a tiny star can be stripped perfectly while a photographer's watermark can't. Spoiler: it's an alpha blend with a fixed, public map.

An alpha blend, not a paste

A normal watermark is a semi-transparent image placed on top, its own colors mixing with the background. The Gemini sparkle is different: a blend of pure white (255) into the corner of your image under a fixed alpha map.

watermarked = alpha × 255 + (1 − alpha) × original

Where the star is solid, alpha is near 1 (pixel → white); where the star is absent, alpha is 0 (pixel untouched); in between, the soft anti-aliased edge.

The alpha map

A grayscale image of the star itself. Read as a 2D array of alphas in [0,1]. Two fixed sizes:

  • bg_48.png — 48×48, both dimensions ≤ 1024, 32px margin.
  • bg_96.png — 96×96, larger than 1024², 64px margin.

Published openly (MIT) by the gemini-watermark-remover project. Not secret — anyone can use them.

Patch position

size   = 96  if (w > 1024 and h > 1024) else 48
margin = 64  if large else 32
x = width  − margin − size
y = height − margin − size

You only touch a size × size rectangle at (x, y). The rest is untouched.

Why it's exactly reversible

The blend is a linear equation in one unknown. Solve:

original = (watermarked − alpha × 255) / (1 − alpha)

Run each channel through it, clamp to [0,255], round. No estimation, no artifacts — except where JPEG already discarded information (small residual).

Threshold & clamping

  • Skip near-zero alpha (< 0.002) — outside the star, divides by ~1.
  • Cap at 0.99 — the star's core is near-opaque; capping avoids dividing by ~0.
  • Clamp [0,255] — rounding can overshoot; clamping keeps a valid pixel.

Why it doesn't apply to real watermarks

A photographer's watermark brings its own colors and partial transparency, and you don't have the alpha map or overlay color. The equation is underdetermined — many originals could produce that watermarked pixel — so you must guess (inpaint, clone, AI-fill). Approximate. The Gemini sparkle is exact because it's a fixed, known, public overlay.

SynthID, separately

SynthID is a different mechanism: subtle perturbations spread across the whole image, no fixed patch, no public map, built to survive editing. Not an alpha blend, nothing linear to invert. That's why the visible sparkle removes and SynthID doesn't.
